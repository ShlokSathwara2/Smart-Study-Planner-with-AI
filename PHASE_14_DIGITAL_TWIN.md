# Phase 14: AI Digital Twin - Personal Learning Model 🧠

## Overview

The **AI Digital Twin** creates a comprehensive, personalized learning profile for each student by analyzing their study behavior, performance patterns, and learning preferences. This digital twin is then used to personalize all AI interactions, making the study planner truly adaptive to individual needs.

---

## What is a Digital Twin?

A **Digital Twin** is a virtual representation of a student's learning journey that includes:

- **Learning Style**: Visual, auditory, reading/writing, or kinesthetic preferences
- **Performance Metrics**: Quiz scores, retention rate, focus score, completion rate
- **Time Patterns**: Best study hours, productive days, session duration preferences
- **Predictive Insights**: Predicted exam score, weak areas, strong areas
- **Revision Patterns**: How often they review, effectiveness of revisions
- **AI Summary**: Natural language analysis generated by Claude

---

## Architecture

### Components

1. **Data Aggregation Layer** (`behaviorAggregator.ts`)
   - Collects data from focus sessions, quizzes, study plans
   - Analyzes time patterns and performance trends
   - Calculates learning style indicators

2. **AI Analysis Engine** (`digitalTwinGenerator.ts`)
   - Uses Claude to generate insights from aggregated data
   - Creates natural language summaries
   - Assigns learning personality labels
   - Generates predictive insights

3. **Context Provider** (`digitalTwinContext.ts`)
   - Formats digital twin data for AI prompts
   - Injects personalization into all Claude API calls
   - Ensures AI responses are tailored to the student

4. **Weekly Scheduler** (`weeklyScheduler.ts`)
   - Automatically updates digital twins every 7 days
   - Tracks update history
   - Handles new users without digital twins

5. **Frontend Profile UI** (`DigitalTwinProfile.tsx`)
   - Beautiful visualization of learning profile
   - Progress rings, charts, and metrics
   - Actionable recommendations

---

## Data Flow

```
User Studies → Focus Sessions, Quizzes, Plans
     ↓
Behavior Aggregator (collects all data)
     ↓
Metrics Calculator (computes scores, trends)
     ↓
Claude AI Analysis (generates insights)
     ↓
Digital Twin Storage (MongoDB)
     ↓
Context Provider (formats for AI)
     ↓
All Future AI Calls (personalized responses)
     ↓
Weekly Auto-Update (keeps fresh)
```

## API Endpoints

### 1. **GET /api/digital-twin/:syllabusId**
Get user's current digital twin profile.

**Request:**
```http
GET /api/digital-twin/abc123?userId=user456
```

**Response:**

## How Digital Twin Enhances AI Calls

### Example: Study Plan Generation

**Without Digital Twin:**
```javascript
// Generic prompt to Claude
"Create a study plan for these topics..."
```

**With Digital Twin:**
```javascript
// Personalized prompt with context
"STUDENT LEARNING PROFILE:
- Learning Personality: Focused Achiever
- Average Quiz Score: 78%
- Best Study Hours: 9:00, 10:00, 11:00
- Weak Areas: Thermodynamics
- Strong Areas: Kinematics

Create a study plan optimized for this student's learning style..."
```

### Implementation

In `studyPlan.ts`:
```typescript
const twinContext = await getDigitalTwinContext(userId, syllabusId);
const digitalTwinPrompt = formatDigitalTwinPrompt(twinContext);

const response = await fetch('https://api.anthropic.com/v1/messages', {
  body: JSON.stringify({
    system: baseSystemPrompt + digitalTwinPrompt,
    // ...
  })
});
```

## Weekly Auto-Update System

### How It Works

1. **Server Startup**: Scheduler initializes
2. **Calculate Next Sunday**: Determines time until midnight Sunday
3. **Set Timeout**: Waits until scheduled time
4. **Run Update**: Processes all digital twins
5. **Skip Recent**: Skips twins updated <7 days ago
6. **Update Old**: Regenerates outdated profiles
7. **Create Missing**: Finds users without twins
8. **Repeat**: Sets interval for weekly execution

### Code Example

```typescript
// In index.ts
scheduleWeeklyDigitalTwinJob();

// Runs every Sunday at midnight
// Updates all digital twins older than 7 days
```

### Caching Strategy

- Digital twins cached for 7 days
- Reuse existing twin if <7 days old
